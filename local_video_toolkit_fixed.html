Local Video Toolkit

Everything runs locally in your browser via ffmpeg.wasm (no uploads). First run downloads the ffmpeg core. Long videos can be RAM-heavy.
This page is opened as a local file (file://).
FFmpeg.wasm uses a Web Worker, and browsers block loading worker scripts from a null origin.
Fix: serve this folder over http://localhost and open it via http.
Windows: python -m http.server 8000
Mac/Linux: python3 -m http.server 8000
Then open: http://localhost:8000/local_video_toolkit_fixed.html
Drop video/audio files here (or use the file picker). Batch processing supported.
Tip: If a file is huge, prefer trimming before resizing.
